One project I’m particularly proud of is my behavior cloning project for an autonomous RC car, where I used imitation learning to train a deep learning model to autonomously navigate a path. To collect data, I manually drove the car around a track while capturing images from a forward-facing camera at around 10 frames per second. Simultaneously, I recorded the steering and throttle angles, which were mapped to the images. This data collection process allowed us to create a dataset where each image was associated with its corresponding control action.

For the deep learning model, I used a **Convolutional Neural Network (CNN)** based on the **NVIDIA end-to-end architecture**. Mathematically, this model uses convolutional layers to apply learnable filters across the image, capturing spatial hierarchies of features like road edges. The extracted features are passed through fully connected layers to predict control actions. During training, the model minimizes the **mean squared error (MSE)** between predicted steering/throttle angles and the actual values, effectively learning the mapping from visual input to control commands.

One important adjustment I made to improve the model’s learning efficiency was incorporating the **ELU (Exponential Linear Unit) activation function**. ELU is particularly useful because, unlike traditional activation functions like ReLU, it can output negative values, which makes it smoother and helps the model converge faster. This allowed the model to better handle difficult scenarios, like sharp turns or sudden lighting changes, by giving it a more stable gradient during training. ELU helped prevent the problem of **vanishing gradients**, which can slow down learning in deeper networks.

To further improve model robustness, I applied **data augmentation** techniques. This included flipping images horizontally, adjusting brightness, and adding random shadows. These augmentations help the model generalize by simulating different driving conditions such as turns, low lighting, or shadows on the road. **Batch processing** was also critical during training, where multiple images were processed simultaneously in batches. This not only sped up training but also allowed for better gradient updates through stochastic gradient descent.

One of the key lessons I learned from this project was the importance of data diversity and preprocessing. Initially, the model struggled with sections of the track that had less training data, which highlighted the need for balanced data across different scenarios. Additionally, I learned how to optimize a deep learning model for real-time control, which is crucial for robotics applications like autonomous driving.
### Issues I ran into
One oddly specific issue I ran into during the behavior cloning project was how the car behaved unpredictably in certain lighting conditions. When driving manually, I collected data in daylight, but during testing, if the lighting changed—like when there were shadows from trees or clouds—the model would start to drift off the track. I couldn’t figure out why it was happening at first, but then it clicked: the model had trouble generalizing to these new lighting conditions because it hadn’t seen enough of that variety in the training data.

I realized the model was essentially overfitting to the exact lighting conditions in which the data was collected, meaning it wasn’t robust enough to handle changes in brightness or shadows. This was frustrating because the car would work perfectly on certain parts of the track, and then suddenly veer off course when a shadow appeared. It wasn’t until I increased the variety of training data—by applying data augmentation techniques like adjusting brightness and adding artificial shadows—that the model became more reliable in different lighting scenarios.

What really ticked me off was how subtle the issue was. Everything worked fine in controlled conditions, but as soon as the environment became more dynamic, the model’s limitations showed. This experience taught me a lot about the importance of training data diversity, especially when working with real-world applications like autonomous driving, where conditions can change rapidly.
