dd### What do you know about web communication standards like HTTP, TCP, UDP, and web sockets
Although I haven’t had hands-on experience working directly with these protocols, I have a solid understanding of what they are and how they function.

- **HTTP (Hypertext Transfer Protocol)** is the standard protocol used for communication on the web. It’s a stateless, request-response protocol where clients make requests and servers send back responses. This is commonly used for transferring data like web pages, API responses, or files.
    
- **TCP (Transmission Control Protocol)** is a connection-oriented protocol that ensures reliable and ordered delivery of data between systems. It's ideal for scenarios where data integrity and accuracy are crucial, like file transfers or certain types of communications where errors can’t be tolerated.
    
- **UDP (User Datagram Protocol)**, on the other hand, is connectionless and prioritizes speed over reliability. It's typically used in real-time applications like video streaming or gaming, where some loss of data is acceptable but low latency is key.
    
- **WebSockets** allow for full-duplex communication over a single connection. Unlike HTTP, which is request-response and stateless, WebSockets enable continuous real-time communication between a client and a server, making them ideal for applications like live chat or real-time notifications.
