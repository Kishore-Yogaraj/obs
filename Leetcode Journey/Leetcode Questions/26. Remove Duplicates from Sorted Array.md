**What does it mean to remove an element in place?**
This basically means to remove the element from the array you're working with and don't create an copy of the array without the specific element you removed

Think about how you can replace an element with another element in the array. In the case of this problem, we will essentially keep the L pointer in one place until there is a difference in value between the index where pointer R is at and the index - 1 of pointer R. If there is a difference in values, we know we have come across a new unique element, so we replace the the point we left the L pointer at and then we move it up. 

R is basically checking for duplicates while L replaces values to make sure all unique values are at the front of the array
### My solution
```python
class Solution:
    def removeDuplicates(self, nums: List[int]) -> int:
        L = 1

        for R in range(1, len(nums)):
            if nums[R] != nums[R-1]:
                nums[L] = nums[R]
                L +=1
        return L
```

### Important Things to Note Here:
The process for this problem can be broken down as follows:

The idea here is to use one pointer to check for duplicates and the other pointer to handle **replacing** elements in the array in place

We first have to start the pointer both at index 1 and not index 0 because the first element will always be unique

We then check to see if the current index of the right pointer is equal to the the index before the right pointer

If they are not equal, we replace the value of the left pointer with the current right pointer and then move the left pointer up

If they are equal, then we ignore the conditional check and keep moving the right pointer

We basically leave the left pointer at a **significant position** and then move the right pointer. The significant position is the position where the value of the element changes, so no repeating elements. We want all the unique elements to be at the front of the array. Therefore we leave the pointer L at the next index to be replaced as long as the R pointer encounter duplicates.

When the R pointer reaches an index where it finds a new unique element then L will be in correct index to bring the unique element to the front.

You then return the index of L which is how many unique elements there are
