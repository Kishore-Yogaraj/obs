### Solution

```python
class Solution:
    def findDuplicate(self, nums: List[int]) -> int:
        slow = 0
        fast = 0

        while True:
            slow = nums[slow]
            fast = nums[nums[fast]]
            if slow == fast:
                break
        
        slow2 = 0
        while slow2 != slow:
            slow2 = nums[slow2]
            slow = nums[slow]
        
        return slow
```

What a crazy problem. You would only know how to solve this problem if you had seen it before. They key here is to us Floyd's cycle detection algorithm which checks for cycles in a linked list like structure.

However, this only works under a very specific case where:
**The size of the integer is n+1 and the range of the values of the integers within the array are from [1,n]. These arrays are also guaranteed to have a duplicate**

For example:
If you have an array: `[1,2,3,4,4]` 
This is a valid array to use Floyd's algorithm because it is an integer of size n + 1 where the range of values inside the integer is `[1,n]` inclusive.

The reason this needs to hold true is because of how we traverse through the array. Typically in a cycle detection problem we would use fast an slow pointer and continue looping through the linked list until we detect a cycle as shown here:
```python
while fast and fast.next:
	slow = slow.next
	fast = fast.next.next
	if fast == slow:
		return True
```

However, there is no way of doing this for an array. Since we are given a very specific array we can use Floyd's algorithm to check for duplicates by infinitely traversing the array similar to a linked list.

This is shown below:
```python
while True:
	slow = nums[slow]
	fast = nums[nums[fast]]
	if slow == fast:
		break
```

Full Implementation:
```python
while True:
	slow = nums[slow]
	fast = nums[nums[fast]]
	if slow == fast:
		break

slow2 = 0
while nums[slow] != nums[slow2]:
	slow = nums[slow]
	slow2 = nums[slow2]

return slow
```
