### The CPU
The CPU is the central processing unit of a computer. It can only do a certain number of things. **It can read and write to memory and it can do math with the numbers they're holding**


> [!Question] What is Memory?
> At its core, memory is a big collection of tiny storage cells that can each hold a number. Each cell is identified by an address (like a house address). Each cell holds a fixed number of bits (binary digits, 0 or 1). The number of bits is typically 8 which is one byte. This means that it can store a number from 0 to 255. The CPU can read and write from these cells.


### Types of Memory Inside a Computer
Not all memory is the same and computers have a hierarchy of memory:
Registers (inside the CPU):
- The fastest memory but very small (a few dozen to hundred). These hold the numbers the CPU is immediately working with
Cache (also inside the CPU, but larger than registers):
- Keeps recently used data close to the CPU so it doesn't have to wait for slower memory
RAM (main memory):
- This is what most people mean by "memory". It's large and relatively fast, but slower than registers/cache. When a program runs, its variables and instructions are stored here

### Executable Program
When we write code, we write a set of instructions that the computer doesn't understand yet. So we use a complier which changes the human readable code into machine readable code. This machine readable code is written in **binary** which are the set of instructions that the computer will use to execute your program. The **CPU** is responsible for stepping through the instructions generated by the compiler as machine code and carries them out one by one. Instructions can include:
- Read these bytes from memory
- Do stuff to them
- Write stuff to memory
- Jump forward this many lines 

**A program expressed in a list of binary instructions is called machine code**

### Stages of compliation

**Lexical Analysis (Tokenization)**
- Breaks the raw text into tokens (smallest meaningful pieces)

```c
int x = 5 + 3;
```

Our tokens are:
int, x , =, 5, +, 3, ;

**Syntax Analysis**
- Checks if tokens follow the rules of the language (grammar)
- Builds a tree structure called the **Abstract syntax tree**
![[Pasted image 20250929123053.png]]
**Semantic Analysis**
- Makes sure the meaning is correct
	- Is x declared before use
	- Do types match
- Adds information like variable types and scope


### Key Idea 
What we need to understand from this is that computers all come down to the **CPU**. The CPU is what reads, writes and performs arithmetic on the instruction you provide to it. Any time you are writing code, you are basically creating instructions to tell the CPU what numbers to write to memory, what numbers to read from memory and what numbers to perform operation on in memory. 

The **Complier** is what's responsible for turning the code we write as human readable to machine readable code.
